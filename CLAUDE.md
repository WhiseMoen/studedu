# StudEdu — контекст проекта для Claude Code

Личное Android-приложение репетитора-студента (Московский Политех):
репетиторское расписание + пары вуза + оплаты учеников, с подсветкой
конфликтов по времени. Владелец — Sapraliev (WhiseMoen на GitHub),
он же единственный пользователь и тестировщик.

## Стек и структура

Kotlin, Jetpack Compose (Material 3), Room (offline-first, источник правды),
Supabase (Postgres+RLS, синк — Этап 5, ещё не подключён), kotlinx-datetime,
dmfs/lib-recur (RRULE), Ktor (расписание вуза), Feather-иконки (compose-icons).

- `domain/` — чистая логика: OccurrenceGenerator (вхождения RRULE-серий),
  ConflictDetector (generic sweep line), LessonVisibilityFilter (скрытие пар).
  Покрыта unit-тестами (`app/src/test`), тесты обязательны при изменениях.
- `data/local/` — Room v3: сущности, DAO, миграции в AppDatabase.
- `data/repository/` — Event/Schedule/Students репозитории.
- `data/schedule/MospolytechProvider` — парсер rasp.dmami.ru (внешний контракт!).
- `data/settings/AppSettings` — SharedPreferences + StateFlow.
- `core/AppGraph` — сервис-локатор (синглтоны), фабрики VM берут отсюда.
- `ui/<фича>/` — Screen + ViewModel (StateFlow, combine).
- `docs/PLAN.md` — бэклог и зафиксированные решения с обоснованиями. ЧИТАТЬ.
- `supabase/migrations/` — SQL для Supabase (0002, 0003 ещё не применены в панели).

## Железные правила проекта

1. Повторяющиеся события НЕ дублируются строками: правило RRULE + исключения.
2. Цена начисления копируется в момент «Проведено» — история неизменна.
3. Скрытые/приглушённые пары НЕ участвуют в конфликтах (слот свободен).
4. Enum'ы в Room хранятся lowercase — как enum-значения в Postgres (для синка).
5. Лимиты монетизации — только через domain/monetization/EntitlementProvider.
6. Неоморфизм — точечно (часы, FAB); списки — плоский пастельный Material 3.
7. Пуш в main после каждой законченной фичи; коммиты по-русски, по фичам.
8. Язык UI — русский. Дизайн: пастель; тёмная тема — Darcula/Obsidian-грейд.

## Как собирать/тестировать

- `gradlew.bat testDebugUnitTest` — юнит-тесты (быстро).
- `gradlew.bat assembleDebug` — сборка APK.
- Пользователь проверяет на эмуляторе «Medium Phone» из Android Studio.
- Машина слабая: не запускать несколько Gradle-задач параллельно.
  В gradle.properties уже стоит kotlin in-process (compile daemon не взлетал).
